# 🖥️ Was-Operating System v4.0![Was-OS Logo](https://img.shields.io/badge/Was--OS-v4.0-blue) ![License](https://img.shields.io/badge/License-MIT-green) ![Offline Ready](https://img.shields.io/badge/Offline-Ready-brightgreen) ![Web-Based](https://img.shields.io/badge/Web-Based-orange) **A fully functional web-based operating system simulation**--- ## 📖 Table of Contents - [About](#about) - [Features](#features) - [Architecture Overview](#architecture-overview) - [File Structure](#file-structure) - [Code Documentation](#code-documentation) - [Terminal Commands](#terminal-commands) - [Installation & Usage](#installation--usage) - [Screenshots](#screenshots) - [Contributing](#contributing) - [License](#license) --- ## 👋 About **Was-OS v4.0** is a lightweight, offline-capable web-based operating system simulation that runs entirely in your browser. It provides a desktop-like environment with window management, a virtual filesystem, and multiple applications—all without requiring any external dependencies or internet connection. Built with pure HTML, CSS, and JavaScript, Was-OS demonstrates how modern web technologies can replicate traditional desktop computing experiences. ### Key Highlights | Feature | Description | |---------|-------------| | 🚀 **Zero External Dependencies** | 100% self-contained, no CDN or APIs required | | 💾 **Persistent Storage** | Files saved via LocalStorage survive browser restarts | | 🪟 **Window Management** | Drag, resize, minimize, maximize, close windows | | 📁 **Virtual Filesystem** | Create, read, update, delete files and folders | | 🎨 **Customizable Themes** | Choose from pre-built themes or upload custom wallpapers | | ⌨️ **Keyboard Shortcuts** | Alt+Tab, Ctrl+W, Escape support | --- ## Initialize VFS (Virtual Filesystem) VFS.init(); // 2. Initialize Core Components (parallel) await Promise.all([WindowManager.init(), Notifier.init()]); // 3. init: (pid, bodyId, state) => { /* ... */ }, cleanup: (pid, state) => { /* ... */ } }); ``` --- ### `js/core/vfs.js` — Virtual Filesystem **Purpose**: Simulates a file system using browser localStorage. **Storage Location**: `localStorage.setItem('wasos_vfs_data', JSON.stringify(data))` **Supported Operations**: | Method | Description | Example | |--------|-------------|---------| | `list(dir)` | List contents of directory | `VFS.list('documents')` | | `read(filePath)` | Read file content | `VFS.read('documents/notes.txt')` | | `write(filePath, content)` | Write/create file | `VFS.write('docs/readme.txt', 'hello')` | | `delete(filePath)` | Remove file | `VFS.delete('trash/old.txt')` | | `exists(filePath)` | Check if path exists | `VFS.exists('pictures/img.png')` | | `isDirectory(path)` | Check if path is folder | `VFS.isDirectory('documents')` | | `move(from, to)` | Move file between locations | `VFS.move('a.txt', 'b.txt')` | | `getAllFiles(dir)` | Recursively get all files | `VFS.getAllFiles()` | | `getStorageUsed()` | Bytes used in localStorage | `VFS.getStorageUsed() / 1024 + ' KB'` | | `clear()` | Wipe all VFS data | `VFS.clear()` | **Default Directory Structure**: ```javascript { documents: { 'welcome.txt': 'Welcome to Was-OS!', 'notes.txt': 'Quick notes...' }, pictures: {}, trash: {} } ``` **Note**: There is NO actual `createDirectory()` method exposed publicly. Directories are created automatically when you write a file with nested path like `'folder/file.txt'`. --- ### `js/core/wm.js` — Window Manager **Purpose**: Handles all window creation, positioning, focus, and manipulation. **Key Properties**: | Property | Type | Default | Purpose | |----------|------|---------|---------| | `windows` | Object | `{}` | All open windows indexed by PID | | `activeWindow` | String | `null` | Currently focused window PID | | `zIndexCounter` | Number | `1000` | Z-index stack counter | | `dragState` | Object | `null` | Current drag operation state | | `resizeState` | Object | `null` | Current resize operation state | **Core Methods**: | Method | Description | |--------|-------------| | `createWindow(pid, title, appId, icon)` | Creates new DOM window element | | `destroyWindow(pid)` | Removes window with animation | | `focus(pid)` | Brings window to front, updates z-index | | `minimize(pid)` | Hides window, marks as minimized | | `restore(pid)` | Shows hidden window, brings to front | | `toggleMinimize(pid)` | Swaps between min/max states | | `maximize(pid)` | Expands to fill workspace (minus panels) | | `close(pid)` | Calls `OS.closeProcess(pid)` | | `getWindowCount()` | Returns number of open windows | | `isMinimized(pid)` | Returns boolean minimized status | | `_cleanupGhostWindows()` | Periodic GC for desynced windows | | `handleDrag(e)` | Mouse drag handler for window moving | | `handleResize(e)` | Mouse resize handler | **Window Limit Protection**: ```javascript if (currentOpen >= this.config.maxWindows) { Notifier.show({ title: 'Window Limit Reached', message: `You have ${this.config.maxWindows} windows opened.`, type: 'warning' }); 🚀 Installation & Usage ### Option 1: Direct File Open (Recommended) 1. **Download/Clone the repository** 2. **Navigate to the folder** in your file explorer 3. **Double-click `index.html`** — opens in default browser 4. **Disconnect internet** — fully functional offline ### Option 2: Local Server ```bash # Using Python 3 python -m http.server 8080 # Using Node.js (npx) npx serve . # Using PHP php -S localhost:8080 ``` Then open: `http://localhost:8080\` ### Browser Requirements | Browser | Version Required | |---------|-----------------| | Chrome | 90+ | | Firefox | 88+ | | Safari | 14+ | | Edge | 90+ | **Required Features**: - ES2022 JavaScript support - localStorage API - Flexbox/Grid CSS - Custom Properties (CSS variables) --- ## 🔐 Privacy & Security Notes | Aspect | Implementation | |--------|----------------| | **Data Storage** | localStorage (browser-only, cleared on manual clear) | | **External Calls** | None (100% offline capable) | | **JavaScript eval()** | Used in calculator (sandboxed within browser context) | | **XSS Protection** | User inputs sanitized where possible, but avoid loading untrusted HTML | **Warning**: The `eval()` in calculator poses minimal risk since it only evaluates user-typed expressions within browser sandbox. --- ## 🎨 Customization Guide ### Adding a New Application 1. **Create file** in `js/apps/myapp.js` 2. **Register app**: ```javascript OS.registerApp({ id: 'myapp', title: 'My App', icon: '🔧', description: 'Custom application', category: 'productivity', singleInstance: false, init: (pid, bodyId, state) => { const container = document.getElementById(bodyId); // Build your UI here }, cleanup: (pid, state) => { // Cleanup resources if needed } }); ``` 3. **Add script tag** in `index.html`: ```html ``` 4. **Refresh page** — app appears in Start Menu ### Changing Startup Apps Edit `js/apps/index.js` order or modify `init.js` startup sequence. ### Modifying Default Files VFS initializes with defaults in `vfs.js`: ```javascript this.data = { documents: { 'welcome.txt': 'Your welcome message...', 'notes.txt': 'Default notes...' }, pictures: {}, trash: {} }; ``` --- ## 🛠️ Troubleshooting | Issue | Solution | |-------|----------| | **Blank screen after reload** | Clear localStorage, refresh page | | **Windows won't open** | Check console for `[WM] WindowManager not loaded` error | | **Files disappear** | VFS uses localStorage—check browser privacy settings | | **Themes not persisting** | Verify `wasos_background` key in localStorage | | **Terminal commands fail** | Ensure `vfs.js` loaded before `terminal.js` | | **Max windows reached (false)** | Reload page or check for ghost windows in developer console | **Debug Mode**: Open browser DevTools (F12) → Console for real-time logs prefixed with: - `[BOOT]` — Initialization messages - `[OS]` — Process/app events - `[WM]` — Window manager events - `[VFS]` — Filesystem operations - `[Notifier]` — Notification logs - `[Terminal]` — Shell activity --- ##
